Project Summary & Goals
What we are building, why it matters, and what success looks like at every checkpoint.
🧩 Project Summary

Brian Goodman Tax & Advisory currently processes complex combined tax return PDFs manually — each file containing a 1040, two-year comparison, 8879, Pennsylvania return, Pennsylvania 8879, and local return — with no structured workflow for extracting or communicating insights to clients. The firm hesitates to use public AI tools like ChatGPT because "I don't know where the information is going." Plus 1 Technology is implementing HatzAI — a private, siloed AI platform — to allow Brian and his team to safely process confidential client tax data using multiple AI models (Claude, ChatGPT, Grok) without exposing information to the internet. The build delivers a prompt library for tax document summarization, form extraction, and tax research in Phase 1; advanced planning and year-over-year analysis prompts in Phase 2; and Canopy API integration plus QuickBooks financial insights in Phase 3. Hard constraints: all AI processing must remain within the HatzAI tenant; no client PII may be processed in free or unmanaged AI tools; Canopy integration requires API access confirmation before Phase 3 build begins.

Business Goals
🔒
Eliminate Data Security Risk
Stop staff from using free ChatGPT or unmanaged Copilot for client tax data. All AI processing runs inside HatzAI's private, siloed environment — no client data leaves the tenant.
⚡
Reduce Tax Return Processing Time
Convert manual PDF interpretation (currently 10+ summaries/day during tax season) into a repeatable 1-click prompt workflow. Brian uploads the PDF, hits the prompt, and gets a clean executive summary ready to send to the client.
📊
Deliver Better Client Communication
Replace raw tax document delivery with clear, digital, client-friendly executive summaries. "Break it down from just the one PDF form… to give the client back a more executive summary so that we can be more digital with them."
🔍
Structured Tax Research with Citations
Replicate Tax GPT's citation behavior inside HatzAI — AI answers tax questions with IRS publication references. Evaluate whether Tax IQ subscription already covers this need to avoid duplicate spend.
💡
Surface Planning Opportunities
Phase 2 builds prompts that review return data and surface potential deductions, planning flags, and advisory opportunities for next year — turning the annual return into a proactive advisory conversation.
🔗
Consolidate AI Into One Platform
Replace fragmented use of free ChatGPT, free Copilot, and separate Tax GPT subscription with a single managed HatzAI tenant. One platform, one cost, all models, no data exposure risk.

Risk Register
AI-specific risks evaluated for this engagement. High-impact items are escalated immediately — not documented and ignored.
R-1 AI Output Accuracy / Hallucination in Tax Summaries Impact: High Likelihood: Medium
Description

LLM generates incorrect numbers, fabricated deduction amounts, or inaccurate form references in tax summaries that Brian sends to clients. Tax data accuracy is non-negotiable — errors create client liability.

Mitigation

Brian reviews every AI-generated summary before client delivery. Prompts are designed to cite page/line sources ("I pulled this from page 8, line 12"). Accuracy threshold defined in UAT before live use.

Contingency

Revert to manual summarization for that return. Engage in prompt re-engineering sprint. Document any errors and use them to improve the prompt. Do not send AI outputs to clients without Brian's explicit review.

Owner

Jessica Peaden (prompt design) / Brian Goodman (review gate)

R-2 Staff Using Free/Unmanaged AI Tools for Client Data Impact: High Likelihood: Medium
Description

"I hesitate sending out tax return or posting tax returns to ChatGPT because I don't know where the information is going." Without a governance policy and clear workflow, Brian or Kevin may continue using free ChatGPT or free Copilot for client data during the build period.

Mitigation

AI governance policy signed by Brian before any client data is processed. HatzAI is the only approved tool. Policy explicitly states what is never to be uploaded to free tools. Communicated to Kevin as well.

Contingency

If free AI tool use is discovered with client data, treat as a data governance incident. Review what was shared, assess exposure, update policy. Accelerate governance policy sign-off.

Owner

Brian Goodman (firm policy) / Marc Umstead (accountability)

R-3 Canopy API Access Delayed or Unavailable Impact: High Likelihood: Medium
Description

Canopy may not expose the API endpoints needed for the integration (client record access, summary attachment, advisory notes). Discovery call confirmed Canopy has an API but specific capabilities are unverified. If API does not support the required operations, Phase 4 integration scope changes significantly.

Mitigation

Marc initiates Canopy API documentation review as soon as Phase 3 sign-off is received. Phase 4 hours are explicitly marked as preliminary pending documentation review. Architecture is designed for this integration only after API is confirmed.

Contingency

If full integration is not possible, fall back to a manual export workflow: Brian copies AI summary and pastes to Canopy manually. Rescope Phase 4 integration hours. Notify Brian of constraint and revised timeline.

Owner

Marc Umstead

R-4 Scope Creep — New Prompt Requests During Build Impact: High Likelihood: High
Description

Once Brian sees what prompts can do, he will likely think of additional use cases ("can you also do a prompt for…"). The discovery call already surfaced QuickBooks insights as a spontaneous addition. Undocumented additions absorb hours that weren't scoped.

Mitigation

Reference the scope document at every check-in. New prompt requests go on a backlog, not into the current build. Anything outside the 5 scoped prompts requires a change request with cost/timeline impact stated before work begins.

Contingency

Document the request and defer to post-Phase 3 or a separate retainer engagement. Marc communicates to Brian that additional prompts are welcome but require scoping — framed positively, not as a refusal.

Owner

Marc Umstead / Jessica Peaden

R-5 Tax Return Format Variability Breaks Prompts Impact: Medium Likelihood: High
Description

Different clients have different return structures — different forms, different states, different number of pages. Prompts built on one sample return may not generalize. Form Extraction prompt designed using section/form-based logic (not page numbers) to handle this, but variability is still a real risk.

Mitigation

Jessica designed Form Extraction using section/form-based logic rather than fixed page numbers per Brian's guidance. Test prompts on at least 2–3 different return types during Phase 2. Edge cases documented and handled before Phase 3 sign-off.

Contingency

Add prompt branching logic for common variations (e.g., no local return included). Document known edge cases in the HatzAI prompt library. Brian flags any return type that doesn't work for a targeted refinement sprint.

Owner

Jessica Peaden

R-6 QBO Migration Delayed — Phase 4 QBO Insight Prompt Blocked Impact: Medium Likelihood: Medium
Description

Brian confirmed migration from QuickBooks Desktop to QBO is expected "within the next year" but is not confirmed. The QBO Financial Insight Prompt requires QBO to be populated with data to be useful. If migration is delayed, this deliverable cannot be built or tested.

Mitigation

Confirm QBO migration status at Phase 3 review. If QBO is live and populated, proceed with the Insight Prompt as scoped. Build Canopy integration first — QBO prompt can be added after migration is confirmed.

Contingency

If QBO migration is not underway by Phase 4 start, defer QBO Insight Prompt to a post-project retainer. Document as deferred in handoff notes. No additional cost to Brian for the deferral.

Owner

Brian Goodman (migration decision) / Marc Umstead (tracking)

R-7 Tax IQ Duplicate Subscription / Overlap with HatzAI Research Prompt Impact: Low Likelihood: High
Description

Brian already subscribes to Tax IQ which includes a tax AI component. If the Tax Research Assistant prompt in HatzAI delivers equivalent quality, there may be a redundant subscription. This is an advisory decision, not a build decision, but should be surfaced clearly.

Mitigation

After Phase 1 demo, run a side-by-side comparison: same tax question in HatzAI Research Prompt vs. Tax IQ AI. Compare citation quality, accuracy, and usability. Present Brian with a clear recommendation.

Contingency

If Tax IQ is better, route tax research there and use HatzAI for document processing only. If HatzAI is equivalent or better, Brian can cancel Tax IQ and reduce subscription costs. This decision has no impact on the build timeline.

Owner

Jessica Peaden (evaluation) / Brian Goodman (subscription decision)
