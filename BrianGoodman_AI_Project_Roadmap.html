Project Summary & Goals
What we are building, why it matters, and what success looks like at every checkpoint.
🧩 Project Summary

Brian Goodman Tax & Advisory processes large combined tax return PDFs — each containing a 1040, two-year comparison, 8879, PA return, PA 8879, and local return — and currently interprets and summarizes this information manually before communicating with clients. Plus 1 Technology is implementing a secure, private AI environment (HatzAI) with a purpose-built prompt library to automate the extraction, summarization, and research workflows that currently consume significant manual time during peak tax season. The expected outcome is that Brian's firm can generate client-ready executive summaries from tax return PDFs in minutes rather than hours, conduct tax research with cited source references, and eventually attach AI-generated outputs directly to Canopy client records without leaving the platform. The hard constraints shaping this build are strict client data confidentiality — Brian has explicitly stated he will not upload tax documents to public AI services — and a lean team size that requires the system to be simple enough for daily use without technical assistance.

Risk Register
AI-specific risks evaluated for this engagement. High-impact items are escalated immediately — not documented and ignored.
R-1 AI Output Accuracy — Tax Return Summarization Impact: High Likelihood: Medium
Description

AI generates incorrect, fabricated, or inconsistent tax summaries that are sent to clients without Brian's review catching the error. Tax errors have compliance and relationship consequences.

Mitigation

Brian-in-the-loop review is mandatory — no AI output goes directly to clients without Brian's approval. Prompts include citation instructions. Phase 1 demo validates accuracy before peak season use.

Contingency

Revert to manual process for affected client. Engage in prompt re-engineering sprint. Document the failure case and add to testing library. Never auto-send to clients.

Owner

Jessica Peaden

R-2 Client Data Exposure via Incorrect AI Platform Use Impact: High Likelihood: Low (mitigated)
Description

Brian or Kevin uploads a client tax return to free ChatGPT, Copilot, or another public AI service out of habit, bypassing HatzAI. This is the exact risk Brian articulated: "I don't know where the information is going."

Mitigation

Governance policy (M1) explicitly prohibits uploading client data to public AI tools. Training covers this. Prompt library makes HatzAI faster and easier than public tools for tax work.

Contingency

If a breach occurs: identify what was uploaded, notify affected clients per firm policy, review governance training. Re-evaluate whether additional access controls are needed.

Owner

Marc Umstead / Brian Goodman

R-3 Canopy API Unavailable or Insufficient for Phase 3 Impact: High Likelihood: Medium
Description

Canopy does not expose the API endpoints needed to attach AI summaries to client records, or the API requires an enterprise tier, manual approval, or custom authentication Brian does not currently have.

Mitigation

Marc to confirm Canopy API capabilities before Phase 3 begins. Do not scope or estimate Phase 3 hours until API access is confirmed. Canopy API review is a Phase 3 entry gate.

Contingency

If Canopy API is unavailable: redesign Phase 3 around a copy-paste or manual upload workflow. Adjust Phase 3 scope and re-estimate hours with client approval before proceeding.

Description

As Brian sees what's possible, he requests new prompts or automations not in the original scope. Without a formal change process, these additions will consume Phase 2 time without additional billing.

Mitigation

Reference the signed SOW at every check-in. New prompt requests go through a change request with a new time estimate before Jessica begins work. Document every out-of-scope request.

Contingency

Owner

Marc Umstead

R-5 Prompt Performance Varies Across Tax Return Formats Impact: Medium Likelihood: High
Description

Different ProSeries configurations and client situations produce different PDF structures. A prompt tuned for a standard 1040 may fail or produce degraded output for a more complex return.

Mitigation

Build extraction logic around form type (not page number) wherever possible. Test against multiple return types in Phase 1. Document known edge cases and build fallback instructions into prompts.

Contingency

For edge-case returns, direct Brian to use a modified or manual prompt. Add failure cases to the prompt library as variant prompts. Phase 2 refinement addresses this systematically.

Owner

Jessica Peaden

R-6 Tax IQ Subscription Overlap Creates Duplicate Tooling Impact: Medium Likelihood: Medium
Description

Brian already subscribes to Tax IQ, which includes an AI tax component. If Tax IQ already covers Tax Research with citations, building a duplicate inside HatzAI wastes build time and creates confusion about which tool to use.

Mitigation

Evaluate Tax IQ capabilities during Phase 1 Week 1. Run the same research query in both Tax IQ and HatzAI prompt. Compare output quality, citation quality, and workflow friction side by side.

Contingency

If Tax IQ is equivalent: deprioritize HatzAI Tax Research prompt build and redirect Phase 1 hours to deeper document summarization work. Brian saves on subscription by not needing Tax GPT.

Owner

Jessica Peaden / Brian Goodman

R-7 HatzAI Platform Pricing or Availability Change Impact: Medium Likelihood: Low
Description

HatzAI is a relatively new platform ("we just started using it a couple months ago"). Platform changes, pricing increases, or discontinuation could affect this build.

Mitigation

Prompts are designed to be model-agnostic and platform-portable. Any prompt built for HatzAI can be run in any other platform. The value is in the prompt library, not the platform binding.

Contingency

Migrate prompt library to alternative platform (e.g., Microsoft Copilot Studio or Claude.ai API). Document all prompts in plain text in SharePoint as platform-independent references.
