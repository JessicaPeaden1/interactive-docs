What we're building & why
After researching Vision ERP's integration capabilities, we confirmed there is no REST API, no CSV import for receiving, and no third-party connector available. This means we can't write directly into Vision. Instead we're building a two-tool automation that handles the heavy lifting — leaving Nicole with a clean report and a much faster entry task.
Koncile
AI Document Extraction Engine
Koncile uses AI-powered OCR combined with large language models to read and understand packing slips — regardless of vendor format, layout, handwritten notes, or complexity. Unlike template-based OCR that breaks when a vendor changes their form, Koncile understands context.
  • Handles any vendor format — zero templates or training required
  • Extracts PO numbers, line items, quantities, prices, tariff lines
  • 95%+ accuracy on complex multi-page documents
  • Confidence scoring flags low-quality extractions for review
  • Reads handwritten notes and poor-quality scans
  • REST API — connects directly into n8n workflow
Why Koncile over alternatives
We evaluated Klippa DocHorizon and Nanonets. Klippa requires template setup per vendor. Nanonets requires training data. Koncile requires zero setup for new vendors — critical with 6+ suppliers including complex tariff-per-line formats.
n8n
Workflow Automation Backbone
n8n is the orchestration layer that connects everything. It watches OneDrive for new packing slips, sends them to Koncile, retrieves the matching PO, runs reconciliation logic, and emails the report to Nicole — automatically within 2–5 minutes of upload.
  • OneDrive trigger watches folder for new PDF uploads
  • Calls Koncile API and parses extracted JSON
  • Matches PO number → retrieves correct PO from OneDrive
  • Line-by-line reconciliation: received vs ordered vs outstanding
  • Tariff line detection and per-job summing logic
  • Generates report and emails to Nicole via Outlook
Why n8n over Make.com or Power Automate
n8n's code nodes give us the custom reconciliation logic needed (fuzzy matching, tariff aggregation, partial receipt tracking). Make.com has limited code support. Power Automate doesn't support Koncile natively.
How they connect
Warehouse → OneDrive uploads packing slip PDF  →  n8n detects new file  →  Koncile extracts all line items, quantities, prices, tariffs  →  n8n matches PO number, retrieves PO PDF, runs comparison  →  Nicole receives clean reconciliation report via email within 2–5 minutes.

Vision remains the system of record. Nicole reviews the report and enters received items faster — with a pre-checked list rather than doing line-by-line comparison from scratch.
