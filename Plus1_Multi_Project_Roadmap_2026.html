How to Use This Multi-Project Engagement Roadmap
Two ways to build this roadmap — pick the one that fits your workflow. Both approaches work for single or multi-project engagements.
🕐 When to Use This Template

Use this template when a client engagement involves two or more concurrent or sequential projects that share a team, a change window, or overlapping timelines. This is your engagement-level planning document — covering everything from the shared change window and cross-project blockers down to individual project milestones, procurement, and next actions. It is designed to be shared with the client and used as the internal delivery reference throughout the engagement.

🗂 How This Template Is Structured

There are two levels of navigation — the Project Switcher (top bar) and the Tab Bar below it.

🏢
Engagement Tab
Shared setup, project map, combined timeline, cross-project blockers, shared procurement, all actions, ClickUp setup, and resources. Fill this in first.
📁
Project Tabs (P1–P4+)
Each project gets its own tab in the switcher with 10 sections: Summary, Scope, Phases, Milestones, Blockers, Risks, Open Items, Procurement, Timeline, and Next Actions.
➕
Adding Projects
4 projects are active by default. Click "+ Add Project" in the switcher to reveal P5 and P6. Maximum 6 projects per engagement.
🔗
Cross-Project Items
Shared blockers, procurement, and actions appear in both the Engagement tab (combined view) and the relevant project tab. Green highlights = cross-project items.
📋 What You Need Before Starting
📞
Completed Discovery
Call notes, pain points, current environment details for each project, and key decisions. Site survey notes if available.
🔲
Agreed Scope (Per Project)
Even rough scope per project — what's in, out, deferred. Flag gaps if uncertain. The template surfaces them explicitly.
📦
Hardware & Procurement Info
Vendor quotes, PO status, lead times — even if not finalized. Flag TBD items explicitly per project.
👥
Team Assignments
Who leads each project, client contacts, key holder per site. Which engineers are shared across projects.
🕐
Shared Change Window
Whether projects share a maintenance window or each need their own. This is often the single biggest cross-project dependency.
🔗
ClickUp Links (Optional)
One ClickUp Space per engagement. Each project gets its own Folder. Enter links in Engagement Setup and per-project ClickUp tabs.
