How to Use This Roadmap Tool
Two ways to generate your project roadmap — pick the one that fits your workflow.
🕐 When to Use

After discovery is complete and the project is moving forward. Use when the engagement involves AI, automation, n8n, APIs, or any intelligent workflow build. This is your scoping and planning deliverable — the document you'd share with the client before any build work begins.

🔗 Entering to ClickUp

Once the roadmap is approved, follow these rules when entering to ClickUp:

Structure
Phases → Create as Folders in the ClickUp Space
Milestones → Create as Tasks inside each Phase folder, tagged as Milestone
Tasks → Created separately using Task Breakdown Prompt output
Time Estimates (milliseconds)
"3600000" = 1h
"7200000" = 2h
"14400000" = 4h
"28800000" = 8h / 1 day
"57600000" = 16h / 2 days
"86400000" = 24h / 3 days
Naming Rules
❌ Don't
"Phase 1: Foundation"
"M1 — Credential Setup"
✅ Do
"Foundation"
"Credential Setup"
🌐 Deploy as a Live GitHub Page

To publish this roadmap as a shareable live URL, use the OneDrive → GitHub Pages n8n workflow:

1. Upload the HTML file to OneDrive with the matching filename
2. Fire the webhook with this body (swap the three values for each new file):

{
  "fileName": "YourFile.html",
  "repoPath": "YourFile.html",
  "commitMessage": "Add your file description",
  "mimeType": "text/html"
}

mimeType stays "text/html" for all HTML pages. OneDrive upload must happen first.
