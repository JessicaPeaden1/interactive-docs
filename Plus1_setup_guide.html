4
Set up your connectors before your first project
Before generating your first roadmap, make sure ClickUp and Microsoft 365 are connected. See Sections 3 and 4 below. If you skip this, Claude will prompt you to connect them mid-workflow — better to do it now.
💡 Pro tip: When you finish a project, use the ⋯ three-dot menu next to the conversation title and select "Move to project" to move it to a client-specific project folder. This keeps your history organized by client.
3
Connect ClickUp
Claude uses the ClickUp connector to add milestones and tasks directly to your project lists. You connect it once — Claude uses it automatically after that.
1
Open Claude Settings
In Claude.ai, click your profile picture or initials in the bottom-left corner, then select Settings.
2
Go to Connectors
In Settings, click Connectors in the left menu. You'll see a list of available integrations.
3
Find ClickUp and click Connect
Scroll to find ClickUp and click the Connect button. You'll be redirected to ClickUp to authorize access.
4
Authorize in ClickUp
Log in to your ClickUp account if prompted. Click Allow on the authorization screen to grant Claude access to your workspace. You'll be redirected back to Claude automatically.
5
Click "Always allow" when prompted mid-conversation
The first time Claude tries to use ClickUp during a project workflow, you'll see a permission popup inside the chat. Always click Always allow — not "Allow once" — so the popup doesn't reappear every session.

4
Connect Microsoft 365
Microsoft 365 gives Claude access to your OneDrive — specifically the Plus1 GitHub Deploys shared folder. This is what triggers the deploy workflow. You also need this for any M365 project work.
1
Go to Settings → Connectors
Same place as ClickUp — Claude.ai → profile → Settings → Connectors.
2
Find Microsoft 365 and click Connect
Click Connect next to Microsoft 365. You'll be redirected to the Microsoft login page.
3
Sign in with your work Microsoft account
Use your Plus 1 work email (the one connected to your Microsoft/OneDrive account). Don't use a personal Microsoft account.
4
Accept Microsoft permissions
Microsoft will ask you to confirm the permissions Claude is requesting — read files, read email, calendar access. Click Accept. You'll return to Claude automatically.
5
Confirm it shows as Connected
Back in Claude Settings → Connectors, Microsoft 365 should now show a green Connected status. If it still shows "Not connected," try clicking Connect again.

5
Access the Shared OneDrive Folder
The Plus1 GitHub Deploys folder is the upload point for all client deliverables. When you drop a file here, Claude picks it up and deploys it to GitHub Pages. You need access to this shared folder.
📬 Ask Jessica to share the folder with you. She'll send you a sharing link by email or Teams. Once you accept, the folder appears in your OneDrive automatically.
1
Accept Jessica's sharing invite
You'll receive an email from no-reply@sharepointonline.com or onedrive.com with a link to the shared folder. Click Open or Accept.
2
Find it in your OneDrive
After accepting, go to onedrive.com and click Shared in the left sidebar. The Plus1 GitHub Deploys folder will appear there. You can also bookmark it directly.
3
How to upload files
Open the folder in your browser. Drag and drop your HTML file directly onto the page, or click Upload → Files. No OneDrive app needed — browser upload works fine.
4
Watch for the (1) rename issue
If a file with the same name already exists, OneDrive will rename your upload to Filename (1).html. This breaks the deploy URL. Always delete the old file first before uploading a new version with the same name.

🚀 How to start your first project
1
Open the Plus 1 Claude project
Go to claude.ai → find Plus 1 Technology — Project Delivery in the sidebar → click New conversation.
2
Add project files first

The agent takes it from there. It will ask if it's an AI or MSP project, confirm your notes, and walk you through every step from roadmap to live URL to ClickUp — without you needing to know what comes next.

When the project is complete, click the ⋯ three-dot menu next to the conversation title → Move to project → select your client project folder. This keeps all history in one place for future sessions.
