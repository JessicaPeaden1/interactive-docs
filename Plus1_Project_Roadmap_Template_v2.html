How to Use This Roadmap Template
Two ways to generate your project roadmap — pick the one that fits your workflow.
🕐 When to Use This Template

Use this template after discovery is complete and the project is moving forward. This applies to any Plus 1 Technology engagement — network, infrastructure, hardware installation, migration, M365, VOIP, security, or mixed projects. This is your scoping and planning deliverable — the document you share with the client and your team before any build work begins.

📋 What You Need Before Starting
📞
Completed Discovery
Call notes, pain points, current environment details, and key decisions from discovery sessions. Site survey notes if available.
🔲
Agreed Scope
Even a rough scope works — what's in, what's out, what's deferred. Flag gaps if uncertain. The template will surface them explicitly.
📦
Hardware & Procurement Info
Vendor quotes, PO status, lead times, ship dates, and warranty terms — even if not finalized. Flag TBD items explicitly.
👥
Team & Site Assignments
Who's building, who's the client lead, who's the key holder. Change window requirements and site access details.
⚠️
Known Blockers & Risks
Open decisions, unresolved scope questions, credential gaps, ISP or vendor dependencies — surface these early.

Staging & Pre-Configuration
Hardware received at Plus 1 staging bench. Pre-configured before arriving on site to minimize installation day downtime.
~[X] daysLow Risk

Deliverables

  • All hardware received and inventoried
  • Firmware updated to approved version
  • Base configuration applied
  • Staging test passed
  • Equipment staged for transport

Key Activities

  • Receive hardware at Plus 1 bench
  • Verify against PO / packing slip
  • Apply firmware updates
  • Apply base configuration templates
  • Bench test — document results

Risk Rationale

  • Low if hardware arrives on schedule
  • DOA unit risk — bench test early
  • Firmware compatibility must be pre-checked
Entry Criteria

Hardware received at staging bench. Credentials confirmed. Config template approved by Plus 1 lead.

Exit Criteria (Testable)

All units pass bench test. Firmware version confirmed. Base config applied and verified. Hardware packed for transport.

Installation & Go-Live

  • Cable management
  • Production config push

Staging complete. Change window confirmed. Key holder on-site. Rollback plan documented and reviewed.

ClickUp Entry Instructions
After roadmap is approved, follow these rules exactly when entering into ClickUp. Naming and time estimate format are critical.

1

Phases → Create as Folders

Each phase becomes a Folder inside the designated ClickUp Space. Do not create phases as tasks or lists.

2

Milestones → Tasks tagged "Milestone" inside each Phase Folder

Each milestone becomes a Task inside its respective Phase Folder, tagged as Milestone. Use the milestone title exactly as written — no prefixes.

3

Tasks → Created separately from Task Breakdown output

Individual tasks come from the Task Breakdown Prompt, not this roadmap. Tasks live as subtasks inside milestone tasks, or in a List inside the Phase Folder.

4

Naming — Clean titles only. No prefixes, numbers, or structural labels.

✅ "Installation"    ✅ "Hardware Staging"

Always pass time_estimate as a quoted string in milliseconds. Never use hours, never pass as a plain number.
